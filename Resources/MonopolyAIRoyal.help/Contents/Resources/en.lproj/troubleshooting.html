Troubleshooting

Common Issues

AI players aren't making real decisions (Mock Mode)

Cause: No valid API key configured, or all keys are rate-limited.

Fix:

  1. Go to Admin > API Keys
  2. Enter a valid OpenRouter API key
  3. Click Validate — ensure it shows "Valid"
  4. Click Save All

API key shows "Invalid"

Cause: The key may be expired, revoked, or incorrectly entered.

Fix:

  • Double-check the key for typos or extra spaces
  • Verify the key is active on your OpenRouter dashboard
  • Try generating a new key

Game is running but no commentary appears

Cause: Commentary is rate-limited or the game speed is too fast.

Fix:

  • Slow down the game speed to Normal or Slow
  • Commentary is generated based on event priority — minor events may be skipped

No sound from narration

Cause: Narrator not enabled, voice provider not configured, or volume too low.

Fix:

  1. Go to Admin > Narrator
  2. Ensure Narrator Enabled is on
  3. Select a provider and voice
  4. Click Test Voice to verify audio output
  5. Check your system volume

ChatterBox server won't start

Cause: Setup script missing, Python environment issue, or port conflict.

Fix:

  • Ensure the chatterbox-setup.sh script is in the Resources folder
  • Check that Python 3 is installed on your system
  • Ensure port 4123 is not in use by another application
  • Try running the setup script manually in Terminal

Budget limit reached — game switched to Mock Mode mid-game

Cause: A budget limit (per-game, daily, or monthly) was exceeded.

Fix:

  • Increase budget limits in Admin > Budget
  • Use cheaper models (Gemini Flash, DeepSeek) to reduce costs
  • The game continues in Mock Mode — no data is lost

ELO ratings aren't updating

Cause: Games may not be completing properly, or database write failed.

Fix:

  • Ensure games end naturally or via the End Game button (don't force-quit the app)
  • Check the Stats > Leaderboard to verify current ratings
  • ELO updates happen after each completed game

Game screen shows blank or unresponsive board

Cause: Game engine may not have started properly.

Fix:

  • Return to the Lobby and start a new game
  • Ensure at least 2 players are selected
  • If the issue persists, quit and reopen the app

Data & Storage

Where is my data stored?

  • Database: ~/Library/Application Support/MonopolyAIRoyal/monopoly.sqlite
  • API Keys: macOS Keychain (secure, encrypted)
  • Settings: Inside the SQLite database

How do I reset all data?

Delete the database file at the path above. The app will create a fresh database on next launch. API keys will be preserved in the Keychain.

Performance Tips

  • Close other resource-heavy applications during streaming
  • Use Instant speed for quick batch games
  • Use Slow speed for the best viewing experience with full commentary
  • The Monte Carlo engine runs 5,000 simulations — this is computationally light but runs async

Still Need Help?

If you're experiencing an issue not covered here, try:

  • Quitting and relaunching the application
  • Checking your internet connection (required for Live Mode)
  • Verifying your API keys are valid and have available credits
