MonopolyAIRoyal Help

Narration & Voice

MonopolyAIRoyal features live text-to-speech narration of game commentary, giving your AI Monopoly games a broadcast feel. Three voice providers are supported.

Enabling Narration

Go to Admin > Narrator and toggle Narrator Enabled on. Select your preferred provider and voice.

Voice Providers

OpenAI TTS

Uses OpenAI's text-to-speech API for high-quality narration.

  • Requires an OpenAI-compatible API key
  • Available voices: Marcus, Aria, Nova, Echo, Alloy
  • Adjustable speed and pitch

ElevenLabs

Premium voice synthesis with natural-sounding AI voices.

  • Requires an ElevenLabs API key (see API Keys)
  • 1000+ available voices
  • Click Fetch Voices to load your available voice list
  • Select a voice and click Test Voice to hear a sample

ChatterBox (Local TTS)

ChatterBox is a fully local text-to-speech server that runs on your Mac. It requires no API key, no internet connection (after initial setup), and supports custom voice cloning from audio samples. It's the recommended provider for privacy-conscious users and for creating unique narrator voices.

Why ChatterBox?

  • Free & unlimited — no per-character costs, no monthly limits
  • Fully local — audio never leaves your machine
  • Voice cloning — upload any audio sample to create a custom narrator voice
  • No API key required

ChatterBox Setup — In-App (Recommended)

The easiest way to set up ChatterBox is directly from the app:

  1. Go to Admin > Narrator
  2. Select ChatterBox as the narrator provider
  3. You'll see the ChatterBox configuration panel with a connection status indicator
  4. If ChatterBox has never been installed, click Install & Start ChatterBox. If already installed, click Start ChatterBox Server
  5. The app runs the bundled setup script automatically. You'll see status updates as it progresses
  6. Wait for the status to show Connected (up to 60 seconds, longer on first install)
  7. Click Fetch Voices to load available voices
  8. Select a voice from the dropdown and click Save All

What the Setup Script Does

When you click the install/start button, the app runs chatterbox-setup.sh which performs these steps automatically:

  1. Detects Python 3.12+ — searches for python3.12, python3.13, python3.14, or python3
  2. Creates a virtual environment at ~/chatterbox-env
  3. Installs Python dependencies — PyTorch, torchaudio, chatterbox-tts, and supporting libraries (numpy, scipy, transformers, etc.)
  4. Clones the API server from github.com/travisvn/chatterbox-tts-api into ~/chatterbox-tts-api
  5. Installs server dependencies — FastAPI, uvicorn, and related packages
  6. Starts the TTS server on http://127.0.0.1:4123
If the server is already running on port 4123, the script detects this and exits immediately without reinstalling anything.

ChatterBox Setup — Manual (Terminal)

If you prefer to run the setup manually, or if the in-app setup encounters issues:

  1. Open Terminal
  2. Navigate to the project folder: 
    cd ~/Desktop/Monopoly/MonopolyAIRoyal-V7
  3. Make the script executable and run it: 
    chmod +x Resources/chatterbox-setup.sh
./Resources/chatterbox-setup.sh
  4. Wait for the installation to complete. You'll see progress output: 
    [1/4] Python: Python 3.12.x
[2/4] Creating virtual environment ...
     Installing dependencies ...
[3/4] Cloning chatterbox-tts-api ...
     Installing server dependencies ...
[4/4] Starting ChatterBox TTS on http://127.0.0.1:4123 ...
  5. Once the server is running, return to the app and click Check Connection in Admin > Narrator

How to Upload a Voice

  1. Ensure the ChatterBox server is running and shows Connected
  2. In Admin > Narrator, find the Upload Voice section
  3. Enter a name for the voice (e.g., "SportsCaster", "DeepVoice", "BritishNarrator")
  4. Click the file picker and select an audio file containing a clear speech sample
  5. Click Upload and wait for the progress indicator to complete
  6. Click Fetch Voices to refresh the voice list
  7. Select your new voice from the dropdown
  8. Click Test Voice to hear a sample narration with the new voice
For best voice cloning results, use a clean audio sample of 10–30 seconds with a single speaker, minimal background noise, and clear enunciation. WAV or FLAC formats preserve the most audio quality.

Finding Community Voices

You can discover pre-made voice models at Fish Audio (fish.audio/discovery). Download voice samples from the community library and upload them to ChatterBox.

ChatterBox Fallback Behavior

If ChatterBox fails during a game (server crash, timeout, etc.), the app automatically handles it:

  1. The current narration attempt is marked as failed
  2. The app falls back to ElevenLabs (if an API key is configured)
  3. If ElevenLabs is unavailable, it falls back to macOS system speech (AVSpeechSynthesizer)
  4. ChatterBox is retried only after a successful connection check

The game continues uninterrupted — narration gracefully degrades rather than failing.

ChatterBox Troubleshooting

"Python 3.12+ is required but not found"

The setup script needs Python 3.12 or newer. Install it via Homebrew:

brew install python@3.12

Or download directly from python.org/downloads. After installing, restart Terminal and try again.

Server starts but status shows "Loading" for a long time

On first launch, ChatterBox downloads the TTS model (~1.5 GB). This can take several minutes depending on your internet speed. Check the Terminal output for download progress. Subsequent launches are instant since the model is cached locally.

Port 4123 is already in use

Another process may be using the port. Find and stop it:

lsof -iTCP:4123 -sTCP:LISTEN
kill -9 <PID>

Then restart ChatterBox from the app.

Installation fails with pip errors

Try running the setup manually with verbose output:

cd ~/Desktop/Monopoly/MonopolyAIRoyal-V7
bash -x Resources/chatterbox-setup.sh

Common fixes:

  • Upgrade pip: pip install --upgrade pip
  • Clear the virtual environment and retry: rm -rf ~/chatterbox-env
  • Ensure Xcode CLI tools are installed: xcode-select --install

Voice upload fails

  • Ensure the file is a supported format (MP3, WAV, FLAC, M4A, or OGG)
  • Check that the server is still running (status shows Connected)
  • Try a smaller audio file (under 30 seconds)
  • Check the ChatterBox admin panel at http://localhost:4124 for error details

Stopping the ChatterBox server

To stop the server, find its process and terminate it:

lsof -iTCP:4123 -sTCP:LISTEN
kill <PID>

Or if you launched it from Terminal, press Ctrl+C in that Terminal window.

Commentary System

The narration system reads commentary generated by the Commentary Engine, which produces ESPN-style play-by-play text. Commentary is prioritized by event importance:

PriorityEvents
CriticalBankruptcy, game over
HighBig rent payments (>$500), major acquisitions
NormalProperty buys, jail events, building
LowDice rolls, small taxes
ColorBetween-turn strategic narrative
At Fast or Instant game speeds, narration switches to template-based commentary to keep up with the pace.
